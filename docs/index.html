Local-first PDF search · no daemon · one binary

Search a folder of PDFs as fast as you can type.

Point pdffff at a folder, it indexes, watches, and searches in milliseconds — literal, regex, or fuzzy — through a Norton-Commander-flavoured terminal UI or a Win95-flavoured desktop window. Results are durable in SQLite, the index lives in RAM, and a debounced watcher keeps both honest as the folder changes.

Three things it does well

Fast

A 65,536-column dense bigram inverted index turns each query into a candidate bitset in microseconds. Verification is exact: memchr::memmem for literal, regex for regex, neo_frizbee for fuzzy. P95 latency over 5,000 chunks: well under 50 ms.

Durable

Extracted text lives in SQLite (WAL, per-corpus DB under your platform's data directory). Second startup re-uses the extracted text — zero re-extraction unless a file's content hash actually changed.

Live

A notify-debouncer-full watcher feeds an arc-swapped overlay (tombstones + overflow chunks) sitting on top of the base index. New, modified, or deleted PDFs surface in search within ~200 ms — no rebuild, no blocking.

Install

One runtime dependency: Poppler's pdftotext. 

# Debian / Ubuntu
sudo apt-get install poppler-utils

# Fedora
sudo dnf install poppler-utils

# macOS
brew install poppler

Then build from source (Rust 1.74+):

git clone https://github.com/lambdabetaeta/pdffff
cd pdffff
cargo install --path .              # TUI binary `pdffff`
cargo install --path . --features gui --bin pdffff-gui

The GUI is optional — gated behind a Cargo feature so the default build doesn't pull in eframe / winit. The Win95-flavoured chrome works the same on Linux, macOS and Windows.

Use

pdffff ~/papers           # interactive TUI
pdffff-gui ~/papers       # Win95-flavoured desktop window

The initial scan and per-PDF extraction run in the background — results appear as soon as the first chunks land in the index, and the status bar shows live progress. Enter opens the highlighted PDF in your host's default viewer and keeps the session open so you can pick more.

Selector mode for shell pipelines

Pass --print-path and Enter exits with the chosen path on stdout instead of opening the file — pdffff then composes like fzf: 

cd "$(dirname "$(pdffff ~/papers --print-path)")"

How it works

pdftotext - - is invoked per PDF (PDF on stdin, text on stdout — the shape rga uses). Output is split on the form-feed (\x0c), normalized through deunicode + lowercase + whitespace-collapse, and sliced into 1,200-character chunks with 200-character overlap. Chunks are persisted to SQLite.

On startup the chunk table is streamed into a flat vector plus a 65,536-column dense bigram posting-list index (adapted from fff's bigram_filter). Queries first ask the index for a candidate bitset — for literal queries this is an AND of consecutive and skip-1 bigrams; for regex an AND / OR tree lifted from regex-syntax; for fuzzy an OR-of-AND of evenly-spaced probe bigrams. Each candidate is then verified exactly. Above the base index sits a small overlay of tombstones and overflow chunks that absorbs file-system events live; arc-swap rebuilds the base out of band when the overlay crosses thresholds, with zero interruption to queries.

Full developer-facing layout in docs/architecture.md.
